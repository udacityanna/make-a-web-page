Lesson 1: HTML Concepts

HTML is the core of the internet and holds every other type of file. It stands for hyper text markup language. A hyperlink is what connects things and makes them 'web-like'.

There are the main pieces; the browser, the internet, and servers using http protocol to communicate.

HTML is made of:
Text content (what you see)
markup (what it looks like)
references to other documents (images or videos)
links

Markup is made up of tags. opening/ closing tags
<tag>; text </tag>

the whole thing is called an element
<b> bold
<em> emphasis (italic)

 If you skip an ending tag, everything after is that state.

HTML attributes are a property of an element.
<tag attribute="value"> contents </tag>

for example the link tag anchor:
<a href="www.reddit.com"> derp </a>

The image tag:
<img src="url" alt="text">
 alt Text is technically required. Nice for blind people.

A VOID tag has no content. img tag has no closing tag
all spaces turn into one space so you need the <br> tag. It's a void tag, too.
<p> is for paragraph. Not void. <p> content </p>

<br> is inline. <p> is block. It makes an invisible box w/ height and width. <b>, <em>, <img> are all inline. So far <p> is the only block element covered.

<span> and <div> are two container tags. The are used for CSS and organization. <Span> is inline and <div> is block

<span class="foo"> text </span>
 <div class="bar"> text </div>

Here is an example html page:

<! doctype html>
<html>

<head>
meta, javascpt, css
<title>Some Words</title>
</head>

<body>
The contents.
</body>

</html>

Computers interpret things literally. Reference material is where programmers get the details.

Lesson 2: If a webpage were a house...

html is the structure (e.g. walls, floor plan)
css is the style (e.g. paint color, type of carpet)
javascript is the appliances (e.g. garage door opener)

All visible elements (e.g. not the head) are rectangular. The page structure is based off boxes.
Document Object Model "DOM" is the tree-like, branching structure of a page. Cross-platform and language independent. The browser creates a DOM tree.

HTML has it's own syntax and rules. A "word" in this language is a tag. A tag is turned into elements using the DOM tree convention.

Tags can have an attribute and a value, too.
<h1 class="article"> content </h1>

Tags can have more than one class.

Lesson 3: Learning Some CSS

This is a CSS call in the head section.
<link rel="stylesheet" type="text/css" href="">
In Sublime Text, if you start typing "li" and then hit tab it fills in the rest for you, except the name of your actual file.

You can also use a <style> tag for small bits of css in an html document. Usually added into the head, but can be inline for odd circumstances. Not best practice.

CSS stands for Cascading Style Sheet and the key word is cascading.

Inheritance: Relies on ancestor-descendant relationship to operate. Mechanism by which properties are applied not only to a specified element, but also its descendants. The enclosing elements pass down. This allows for less repetition and faster page loading.

In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing.

Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.

Given the following style sheet:
h1 {
color: pink;
}

Suppose there is an h1 element with an emphasizing element (em) inside:
<h1>
This to <em> illustrate </em> inheritance
</h1>

If no color has been assigned to the em element, the emphasized word "illustrate" will inherit the color of the parent element, h1. The style sheet h1 has the color pink, hence, the em element will likewise be in pink.

Selector: what your style will apply to.
a tag
a class (.description)

Learn more at CSS tricks. Or alternately Mozilla Developer Reference.

Lesson 3 1/2: Combine HTML and CSS

Using the right HTML tag is important for organizational and searchability reasons.

Box model: Margin > border > padding > content

Padding is affected by the background color of the box.
Border color inherited from color property of the box.
Margin has no color: it's completely transparent.

Size of the element is equal to the border, padding, and content widths.

Box-sizing: border-box; is made to include these together, so that you can calculate layout easier, but it's a new property and must have a browser specific prefix.

Percentages are flowable sizes. Use a max width. Pixels are static.

Alignment and positioning in CSS is one of the most difficult concepts. Box-sizing: border-box; makes knowing the total dimensions easier. Flex box makes flexible positioning easier.

By the end of this lesson I should know: HTML elements are boxes, and each box has 4 components. Because of how they can interact, it can be hard to get the sizes of boxes right. Using percentages rather than absolute pixels can help as well as setting the box-sizing attribute to border-box for every element. Some of these display differently in different browsers, so that you may need prefixes.

Since divs are block elements (not inline) by default they take up the entire width of the page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let them sit beside each other.

Tips for Testing

You can use this CSS to display a red border around all the boxes for layout purposes.
* {
outline: 1px solid red !important;
}

 First work on the structure by "boxifying". Then add tags. Then look for repeated styles and elements. Write HTML. Then apply styles from biggest to smallest.

Then fix things and test. Change browsers, resize windows, etc. Browser dev tools are a great help.
